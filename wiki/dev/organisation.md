# Organisation et bonnes pratiques du dossier dev

Le dossier `dev/` regroupe les développements spécifiques, composants et extensions personnalisés du site kd-com.

## Structure recommandée
- Un fichier par composant ou fonctionnalité personnalisée.
- Documentez chaque composant avec son objectif, sa structure (HTML, SASS, JS), son fonctionnement et des exemples.
- Ajoutez les références aux fichiers sources (SASS, JS, PHP) pour faciliter la maintenance.
- Pour les effets AOS, centralisez la documentation sur l’utilisation des classes CSS (`aos-*`, `aos-delay-*`) et la configuration via les scripts JS du thème.

## Bonnes pratiques
- Centralisez la documentation technique personnalisée dans ce dossier.
- Utilisez des noms explicites pour chaque fichier et composant.
- Mettez à jour la documentation à chaque évolution du code.
